# 插件系统更新计划：视图-动作协议与能力发现机制

## 1. 核心背景与问题
当前插件系统存在“有操作接口、有 UI 渲染，但缺失调度逻辑”的问题：
1.  **UI 交互缺失引导**：前端渲染出卡片后，不知道点击卡片应触发哪个后端操作（如跳转详情、打开配置）。
2.  **系统调度硬编码**：系统组件（如全局邮箱）依赖特定插件（如 `agent_manager`），导致耦合度高，无法动态替换或扩展。

## 2. 解决方案概览
引入 **BFF (Backend for Frontend)** 设计思想，通过元数据驱动前后端交互：
*   **视图-动作协议 (View-Action Protocol)**：后端在返回数据（View）时，同时下发交互指令（Action），前端通过通用调度器执行指令。
*   **能力发现机制 (Capability Discovery)**：通过标准化的“能力标签”与“接口契约”，实现系统组件对插件的动态发现与调用。

---

## 3. 详细实施计划

### 第一阶段：UI 交互闭环 (Agent 管理器视图)
**目标**：实现 Agent 管理器从“卡片列表”到“配置详情”的无缝跳转，无需前端硬编码路由。

#### 后端 (Backend)
1.  **Schema 升级**：
    *   在 `CardItem` 模型中新增 `actions` 字段，用于描述点击等交互行为。
    *   定义 `Action` 结构：`{ type: "invoke_operation" | "link", operation: string, params: dict }`。
2.  **BFF 操作实现**：
    *   `get_card_view`：获取 Agent 列表，返回 `RenderType.CARD`，每个卡片携带 `click` 动作指向 `get_agent_detail`。
    *   `get_agent_detail`：获取单个 Agent 详情，返回 `RenderType.CONFIG`（配置表单）或 `RenderType.DETAIL`。
3.  **元数据配置**：
    *   在 `agent_manager` 插件定义中，配置 `data_source_entry_point="get_card_view"`，作为插件的默认入口。

#### 前端 (Frontend)
1.  **通用调度器 (Dispatcher)**：
    *   在插件渲染组件中实现 `handleAction(action)`。
    *   支持 `invoke_operation`：自动发起 `POST /plugin/{id}/operation/{op_name}` 请求，并根据返回的新 `RenderType` 刷新区域。
2.  **视图组件增强**：
    *   更新 `CardView` 组件，使其响应 `actions.click` 事件。

### 第二阶段：系统能力调度 (邮箱模块)
**目标**：解耦系统邮箱组件与具体插件实现，支持未来替换邮件服务提供商。

#### 后端 (Backend)
1.  **能力标准定义**：
    *   定义 `capability:mail` 的接口契约（输入：`{recipient, content}`, 输出：`{success, message}`）。
2.  **插件能力声明**：
    *   在 `agent_manager`（或未来的邮件插件）的 `tags` 中添加 `capability:mail`。
    *   确保其实现了标准的 `send_mail` 操作。

#### 前端 (Frontend)
1.  **服务发现 (Service Discovery)**：
    *   在 `pluginService` 中新增 `findPluginByCapability(capability_tag)` 方法。
2.  **业务解耦**：
    *   重构 `MailContext`：初始化时查找拥有 `capability:mail` 的插件 ID。
    *   发送邮件时，动态调用该插件的 `send_mail` 操作。

## 4. 协议设计详情 (Draft)

### 4.1 Card Item with Actions
```json
{
  "id": "agent_1",
  "title": "小说助手",
  "summary": "协助撰写小说大纲与正文",
  "actions": {
    "click": {
      "type": "invoke_operation",
      "operation": "get_agent_detail",
      "params": { "agent_id": "agent_1" }
    },
    "menu": [
        { "label": "启用", "operation": "enable_agent", "params": { "id": "agent_1" } },
        { "label": "删除", "operation": "delete_agent", "params": { "id": "agent_1" }, "danger": true }
    ]
  }
}
```

```python
@plugin_meta(

    # ...

)
class AgentManagerPlugin:
    # ...
```
