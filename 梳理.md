插件系统到前端的接口映射的问题

本质上是一个内部系统如何和一个外部系统进行交互的过程

所以可以抽象文

某个系统到某个系统的映射问题

[系统A:接口]-json->[系统B:接口]

可以看成是两个系统通过一个标准化的语言(这里是json)和一个协议(http)进行信息的交换,这份约定俗称是人为规定的,所以可以,

但是现在是这样子的

[系统A:接口]-json->[系统B:接口(不确定)]

在这里,系统A的接口动态的产生变化,而系统B只能获取系统A有哪些接口,但是却不知道这些接口和本系统之间的映射关系

是的,问题不是在于接口发现,而是接受方在没有人为一一对应的情况下如何完成外部系统到内部系统的映射

那果然只有通过契约的方式了,也就是规定一个可以从系统A到系统B内部组件映射的的协议...(原来这就是协议吗?就是规定两个不同的系统之间如何将自身的内部资源进行交互)

如果从架构的上出发,我首先会预定义好一系列的前端组件,也就是所谓的`插槽`或者说`半插槽`,之后后端提供一些的映射包,在后端的接口返回的时候,后端使用这些映射包进行数据模型的处理,将数据模型映射到前端的插槽中,前端根据插槽去渲染整个页面。

协议:
前端的组织方式:
页面(前端组织插槽): 插槽(前端固定位置,后端申明使用什么插件): 组件(前端定义,后端映射)

后端:
使用链式的方式申明组件,即
class PageName(Pgae):
    class SlotName(Slot):
        class ComponentName(Component):
            def __init__(self, somescham):
                self.somescham = somescham
            

使用时
每个操作返回都使用上面的内容进行返回
例如
class Component():
    @class
    def with(self,**kargs):
        pass

class Home(Pgae):
    class Card(Slot):
        class Item(Component):
            ...
class Home(Pgae):
    class CardDetails(Slot):
        class Info(Component):
            def __init__(self, name,weight):
                self.name = name
                self.weight = weight
@operation(
    with_ui = [Home.Card.Item.with(name="item")],
    interface = "/get_page"
    ui=Home.CardDetails.Info
    ) # 这里表示的意思就是绑定这个操作到Home.Crad.Item并name="item"的组件上(类似于条件绑定),其使用接口"/get_page"来访问这个组件,并将返回值映射到Home.CardDetails.Info组件中
def get_page(card_id:str):
    return {"name":"Pgae","weight":0}
这里将 return值映射为Info的schame->如果有不匹配的,则存储到**kargs中
表示,如果调用了/get_page这个接口,就会渲染到表示渲染到Home.Card插槽中的Info容器中

后端会收集一份信息,并提供一个接口给前端使用(展示假设系统的插件全部都使用)
接口:home/card/item?name=item/get_page
