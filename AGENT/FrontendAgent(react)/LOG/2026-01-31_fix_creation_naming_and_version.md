# 变更记录: 修复新建章节命名与版本显示问题

- **时间**: 2026-01-31 15:00
- **目标**: 修复在作品详情页创建新卷或章节时，用户输入的名称被忽略以及初始版本号显示不规范的问题。
- **变更范围**:
  - `src/app/works/[id]/page.tsx`:
    - `handleCreateVolume`: 增加 `name` 参数，传递给 API 和本地状态。
    - `handleCreateChapter`: 增加 `name` 参数，传递给 API 和本地状态。
    - 优化乐观更新逻辑：创建新章节时，显式设置 `currentVersionName: 'v1.0.0'` 和 `versions` 中的 `version: 'v1.0.0'`，确保与编辑器保存后的行为一致。
- **验证方式**:
  - 在目录栏点击“+”或右键菜单创建新卷/章节，输入自定义名称（如“第一章 序幕”）。
  - 确认创建后的节点直接显示输入的名称，而非“新建章节”。
  - 确认新建章节的版本标签显示为 `v1.0.0` 而非 `v1` 或 `undefined`。
- **结果**:
  - 新建操作的所见即所得体验得到修复，消除了页面间的数据显示差异。
