# 结构
├── PROJECT/
│   ├── DOCUMENTS/          # [文档中心] 存放项目的重要文档、设计稿、流程图等。
│   │   └── README.md       # 文档目录说明
│   ├── TASKS/              # [任务中心] 存放任务分配、进度跟踪及各个Agent的任务文档。
│   │   ├── README.md       # 任务目录说明
│   │   └── TASK.md         # 任务模版或具体任务描述
│   ├── SPECIFICATION.md    # [规格说明] 项目规格文档,描述项目基础规范、技术选型、代码规范等。
│   ├── LOG.md              # [项目日志] 项目操作日志,记录项目进度日志、关键操作等。
│   └── README.md           # [本文件] PROJECT目录的说明文档。

# 详情
PROJECT(或类似于的内容): 
    描述: 项目资源，该资源描述了Agent和人类应该如何正确的开发当前项目，使项目的发展有迹可寻。是整个资源中权限等级最高，不允许普通的，未经许可的Agent对该资源进行任何的操作。否则系统或人类将会选择移除该Agent以确保项目的稳定进行。
    
    结构: PROJECT的核心组成部分
        
        文档中心(DOCUMENTS/):
            描述: 存放该项目中中重要的或通用的或已经过管理员审核的从Agent内部拉取下来的文档。
            作用: Agent可以根据这些文档进行项目的分析、设计和实现。
            注意: 一般的Agent不被允许直接操作PROJECT/DOCUMENTS资源中的文档。
        
        任务中心(TASKS/):
            描述: 存放该项目中的所有任务文档。包括每个Agent在各个阶段需要完成的任务。
            作用: 这里的任务文档是项目进行阶段性演化的依据。每个任务都有清晰明确的可验收的任务描述和验收指标。
            流程: Agent领取任务 -> 理解需求 -> 创建/更新任务文档 -> 执行任务 -> 标记完成。
        
        规格文档(SPECIFICATION.md): 
            描述: 项目的基础规范文档。
            作用: 指导Agent以怎么样的组织形式和写作方式进行项目的开发和团队的协助。包含技术选型、代码规范、架构定义等。
            原则: 当Agent的代码不合格或尝试与原本代码冲突时，Agent将重新阅读规格文档以确保Agent能以标准的内容进行创作。

        项目日志(LOG.md): 
            描述: 由系统或人类进行记录，标记着和主项目紧密相关的进度汇报。
            作用: 记录项目本身的进度日志，辅助Agent理解当前项目的状态和隔离任务。包括代码模块正式进入主项目的关键内容、重要报告等。
            
    补充: 项目资源是项目推进的重要依据。每个Agent都有责任确保项目资源的正确性与安全性。当项目中存在项目资源混乱或被破坏时，每个Agent有权向管理员提出警告并不参与任何的项目开发。
