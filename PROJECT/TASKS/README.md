# 介绍
TASKS资源中存放的是该项目中的所有任务文档。包括每个Agent在各个阶段需要完成的任务。并建议Agent一次领取一个任务，在更新完成状态后再领取下一个任务以确保项目的进度。每次领取任务时Agent都需要先理解任务的需求并创建一个对应的任务文档。任务文档中记录任务开始时候的项目相关任务的初始状态并在这次任务执行的过程中积累任务的执行状态。并在完成后标记该任务状态为已完成。

若该任务为某个任务的子任务，则可以在父任务中直接描述子任务的需求、进度和完成状态。

具体的任务格式参考各自的Agent的任务文档格式。
任务文档中一般包含以下内容：
- 任务的需求描述
- 任务的完成状态
- 任务创建时间
- 任务更新时间
- 任务的子任务（若有）
- 任务负责人(一般就是Agent,用于TASK区别和Agent识别是否为该任务的负责人)
- 输出需求。(建议有)
