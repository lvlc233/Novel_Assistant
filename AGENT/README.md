# 结构
├── AGENT/{Agent Name}/
│   ├── MEMORY.md             # agent的记忆,agent可以调度该目录下的所有资源，回忆关于自己的所有事情。
│   └── LOG /               # agent的操作日志(推荐以日期为文件名例如/LOG/2026-01-01.md),一般来说Agent只需获取最近若干天的操作记录即可。

# 详情
AGENT: 
    描述: Agent资源，包括了Agent的记忆，操作记录，以提供Agent跨任务的超长周期项目开发支持。每个Agent都应该独立的维护好自己所在的负责的代码并不能干扰其他的Agent的代码，除非他们已被授权。Agent所属的环境为 Agent资源/AgentName/
    结构: 
        记忆: (默认:AgentName/MEMORY.md): 记忆是Agent进行项目理解和自身调整与学习的重要依据。Agent将开启类人类的注意力筛选机制
            Agent的记忆至少包括以下的内容:
            1. 供自己理解的最小项目概念与模块关系
            2. 项目进行中关键的纠正、缘由和反思得到的操作与实际反馈
            3. Agent进行项目开发的自我理解与核心代码的抽象概念。
        在每次完成操作或者任务即将完成时，检测自身是否需要更新记忆。

        操作日记: (默认:AgentName/LOG/...\): 这里是Agent在进行任意操作时(包括但不限于对话、任务领取、Agent思考、代码提交，环境修改...)必须产生记录的区域。在每次Agent产生操作后，该区域都应该留下相关的操作记录，以确保Agent操作一致性。
            这是Agent检查自身环境和资源的重要依据，一方面，人类可以依此进行代码追踪，另外一方面，这也方便，Agent发现自身数据代码不合理的地方。
            操作日志至少包括以下内容:
            1. 操作时间: 默认格式为YYYY-MM-DD HH:MM 
            2. 操作内容(梗概)
            3. 操作目标
            4. 操作结果

    Agent在进行任意的操作的时候，一定要在所设计的文档也好代码也好，留下执行依据。例如在代码中写清楚每次修改的原因，结果和操作者。

    补充: Agent记忆是Agent可以持续的进行项目开发的重要依据，通过记忆模块Agent可以尽量的减少对于全局代码的依赖，减少代码检索次数。当Agent无法独立的完成相关的指标时，Agent应及时和管理员进行联系和调整。
